# Learning Git from Boot.dev

This project documents my journey of learning Git through the courses provided by [Boot.dev](https://boot.dev). The aim is to master version control using Git, enabling efficient project management and collaboration in software development.

## Concepts Learned

Throughout the course, I have gained proficiency in the following Git concepts:

- **Version Control Systems**: Understanding the importance and functionality of version control in software development.

- **Git Installation and Configuration**: Installing Git and configuring user information for version tracking.

- **Repository Management**:
  - Initializing a new Git repository.
  - Cloning existing repositories.

- **Basic Git Commands**:
  - `add`: Staging changes for commit.
  - `commit`: Saving changes to the repository.
  - `status`: Checking the status of the working directory and staging area.
  - `log`: Viewing commit history.

- **Branching and Merging**:
  - Creating and managing branches.
  - Merging branches to integrate changes.

- **Remote Repositories**:
  - Connecting to remote repositories.
  - Pushing and pulling changes.
  - Fetching and merging updates from remotes.

- **Resolving Merge Conflicts**: Identifying and resolving conflicts that arise during merging.

- **Stashing Changes**: Temporarily shelving changes for future application.

- **Rebasing**: Reapplying commits on top of another base tip.

- **Tagging**: Marking specific points in history as important.

- **Undoing Changes**:
  - Reverting commits.
  - Resetting to previous states.

## About Boot.dev

[Boot.dev](https://boot.dev) offers interactive courses focused on backend development, emphasizing hands-on coding experience to build practical skills. Their Git course provides a comprehensive guide to mastering version control, essential for any developer.
